1 <h1>Remote Instrumentation with Cuke4php</h1>
2     <p>At times, it is helpful to be able to interact directly with backend code on a server being tested using Selenium or Watir.</p>
3     <h2>How does it work?</h2>
4     <ul>
5         <li>Cucumber process on local machine asks remote machine about step definitions it doesn't understand</li>
6         <li>cuke4php_server listener on remote machine on a given port will attempt to handle unknown steps</li>
7     </ul>
8 
9     <h2>Why?</h2>
10     <ul>
11         <li>Create/destroy objects when no UI is available</li>
12         <li>Setup/teardown of server state</li>
13         <li>Alter configs</li>
14         <li>Create/destroy accounts (avoid CAPTCHAs)</li>
15         <li>Trigger events behind the scenes</li>
16     </ul>
17     
18     <h2>Caveats</h2>
19     <ul>
20         <li>Concurrency of tests can be tricky</li>
21         <li>Concurrent use by humans can be a problem</li>

23     </ul>
